# Diagnostic Objects

The full local workflow generates ArviZ NetCDF posterior and
posterior-predictive objects during execution.

These binary diagnostic objects are intentionally excluded from
the public GitHub repository to keep the portfolio repository
lightweight and practical to clone.

The public repository retains:

- the complete Colab notebook;
- convergence-diagnostic CSV summaries;
- posterior summaries;
- posterior-probability tables;
- prior-sensitivity summaries;
- posterior-predictive audit tables;
- and publication-style diagnostic figures.

To regenerate the NetCDF objects, run the notebook sequentially
in Google Colab.
